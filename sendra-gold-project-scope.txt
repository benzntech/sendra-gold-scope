PHASE 2 — DAR API REVERSE ENGINEERING (21 Jun)
-----------------------------------------------
- Cracked darjewellery.com's product API at POST /api/product/productListPage
- Discovered double-wrapped JSON body: data → stringified JSON
- Fields: currency, pc.dcount (48max), pc.page, url, client_id(233), jbo_id(233)
- Image URLs base64-encoded in "Images" field
- Alternative: thumbnail_image[0] gives direct URL
- Extracted 230+ category URLs from sitemap
- Sample: Gold Earrings alone = 15,273 products, 319 pages

PHASE 7 — BUG FIXES (23 Jun)
-----------------------------

  Issue: Blank images
  Root cause: Import stored images at
    storage/app/public/app/public/product/ID/
    instead of
    storage/app/public/product/ID/
    (extra "app/public/" nesting in path)
  Fix: Copied 37,261 product directories to correct path
       Images now serve correctly at /storage/product/ID/filename.jpg
  Status: Resolved ✅

  Issue: Product page 404
  Root cause: Tested invalid URL keys like "gold-chain-1"
  Reality: Valid product keys (e.g. "22-kt-gold-nethi-chutti-00227104")
           work fine through SSR GraphQL
  Status: Not a bug ✅

  Issue: Storefront GraphQL error
  Root cause: Bagisto uses Relay-style pagination
              (first/after/edges/node/pageInfo/totalCount)
              NOT simple pageSize/items pattern
  Fix: SSR configured to hit :8000/api/graphql directly
       Browser proxies through /api/graphql Next.js route
  Status: Resolved ✅

PHASE 8 — BROKEN LINK FIX + SEARCH REDIRECT (24 Jun)
-----------------------------------------------------
  Issue: Quick shop links (e.g. /search/chain) returning 404
  Root cause: Category slugs are 3-level deep (e.g. "traditional-silver-jewellery-
              and-articles-silver-women-jewellery-chain") but frontend links use
              short names (e.g. "chain")
  Fix: Added fallback redirect in /search/[collection]/page.tsx — if exact
       category slug not found, redirects to /search?q=<slug>
  Also: E2E test suite set up at benzntech/sendra-e2e-tests (Playwright + pytest)
  Status: Resolved ✅
